# Known R-parity gap signatures (the fuzzer's --baseline allowlist).
#
# Each line would be a gap-CLASS signature: the diverging probe with numeric
# literals masked to N/F and quoted words to W, so many instances collapse to
# one class.
#
# Usage:  parity-fuzz --count 5000 --baseline tests/data/parity_fuzz_baseline.txt
# A run exits 0 while every divergence is in this list; a NEW gap-class (a
# regression, or a newly reached surface) exits 1 so CI/you can triage it.
#
# This list is EMPTY: every gap class the fuzzer once found across its 21
# surfaces has been fixed, and repeated multi-seed sweeps now report zero
# divergences. A non-empty run is therefore a real regression to investigate.
#
# Deliberately NOT generated (so they never appear here): `%%`/`%/%` and `var`
# on values at or beyond f64's exact-integer range — R computes those in
# extended precision (C `long double`), which Rust's f64 cannot bit-reproduce,
# and R itself flags the modulus case as "probable complete loss of accuracy".
# The generators avoid feeding such precision-extreme operands into those ops.
